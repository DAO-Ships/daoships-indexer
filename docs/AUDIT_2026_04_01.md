# Indexer Audit Report — 2026-04-01

Full codebase audit after INavigator integration, ragequit total_shares fix, navigator.metadata removal, and navigator.allowlist addition.

---

## HIGH

### H1. Reorg cleanup misses total_shares / total_loot recalculation — FIXED

**File:** `supabase/migrations/schema.sql` (ds_delete_events_after_block)

`ds_delete_events_after_block` recalculates `proposal_count` and `active_member_count` from surviving rows, but `total_shares` and `total_loot` are delta-accumulated (handleMintOrBurn, handleRagequit) and never recomputed. After a reorg, the deltas replay against stale totals, causing double-counting.

**Fix applied:** Added `total_shares` and `total_loot` recalculation via `COALESCE(SUM(m.shares/m.loot), 0)` from `ds_members` to the UPDATE in `ds_delete_events_after_block`.

## MEDIUM

### M1. Transfer topic0 over-fetching in fetchAllLogs — FIXED

**File:** `src/services/processor.ts`

`fetchAllLogs` used topic0 OR-filter with no address filter. Transfer's topic0 (`0xddf252ad...`) matches every ERC20/ERC721 transfer on the chain. The RPC returned all matching logs, then the indexer filtered in-memory to known addresses.

**Fix applied:** `fetchAllLogs` now passes `knownAddresses` as the server-side address filter to `getLogs`. In-memory filtering removed. Comment notes batching needed if address list exceeds RPC limits (~50-100).

### M2. Reorg recovery doesn't clean ContractRegistry — FIXED

**File:** `src/registry/contract-registry.ts`, `src/index.ts`

After reorg cleanup, the SQL deletes orphaned navigators/DAOs but the in-memory `ContractRegistry` retained stale entries.

**Fix applied:** Added `clear()` method to `ContractRegistry`. Reorg handler in `index.ts` now calls `registry.clear()` then rebuilds from surviving DB rows via `getAllDaosIterator` and `getActiveNavigatorsIterator`.

### M3. Member upsert on Transfer missing updated_at — FIXED

**File:** `src/handlers/tokens.ts`

The `upsertMember` calls in the Transfer handler didn't include `updated_at`. PostgreSQL's `DEFAULT NOW()` only applies on INSERT, not on the UPDATE branch of an upsert.

**Fix applied:** Added `updated_at: now` to both sender and receiver upsert payloads in `handleTransfer`.

## LOW

### L1. treeDump stored without deep sanitization — FIXED (downgraded from HIGH)

**File:** `src/handlers/poster.ts`

`treeDump` passed through with only a type check. The real risk is only if a frontend naively spreads the object.

**Fix applied:** Added `sanitizeJsonb()` that recursively strips `__proto__`/`constructor`/`prototype` keys and caps depth at 5 levels. Applied to `treeDump` in `validateNavigatorAllowlist`.

### L2. NavigatorDeployed multi-log warning missing — FIXED

**File:** `src/handlers/daoship.ts`

If a navigator somehow emits multiple `NavigatorDeployed` events, the handler silently takes the first.

**Fix applied:** Added `logger.warn` when `logs.length > 1` before processing the first log.

### L3. Unused variable memberId in handleConvertSharesToLoot — FIXED

**File:** `src/handlers/daoship.ts`

**Fix applied:** Deleted the unused `const memberId = makeMemberId(daoId, from)` line.

### L4. Type mismatches: NavigatorRow.paused, navigator_type — FIXED

**File:** `src/types/index.ts`

**Fix applied:** Changed `paused: boolean` to `paused?: boolean` and `navigator_type: string` to `navigator_type: string | null`.

### L5. schemaVersion gate accepts any truthy value — FIXED

**File:** `src/handlers/poster.ts`

**Fix applied:** Changed `!parsed.schemaVersion` to `typeof parsed.schemaVersion !== 'string'`.

### L6. daoAddress not validated as hex in profile validators — FIXED

**File:** `src/handlers/poster.ts`

**Fix applied:** Added `ETH_ADDRESS_RE.test()` checks to `validateDaoProfileInitial`, `validateDaoProfile`, `validateDaoAnnouncement`, and `validateMemberProfile` (optional for member.profile since daoAddress is optional there). Moved `ETH_ADDRESS_RE`/`BYTES32_RE` constants above validators.

### L7. permissionToLabel masks bits 3-7 silently — FIXED

**File:** `src/handlers/daoship.ts`

**Fix applied:** Tightened handler validation from `permission > 255` to `permission > 7`. Events with reserved bits (3-7) are now rejected with a warning log. Test updated accordingly.

### L8. rawCall has no timeout — FIXED

**File:** `src/services/blockchain.ts`

**Fix applied:** Wrapped `provider.call()` in `Promise.race` with a 10-second timeout.

### L9. Redundant DAO fetch in profile.initial handler — FIXED

**File:** `src/handlers/poster.ts`

**Fix applied:** Changed `determineTrustLevel` to return `{ trust, dao }`. The `profile.initial` case now uses the pre-fetched DAO from trust determination instead of a redundant `getDao` call.

### L10. Missing tests — FIXED

**Fix applied:** Added two tests:

- `poster.test.ts`: "member.profile rejected for UNTRUSTED user (non-member)" — verifies trust boundary
